# 使用者環境與限制

## 使用平台
- claude.ai 免費版網頁（主要）及手機 App
- 有 Claude Project 功能，上限 5 個
- 無法跨 session 保留記憶，每次需貼入 handoff.md
- 不使用 Claude 桌面版、本機 AI agent

## 本機環境
- Windows 10 + PowerShell 7
- Git 已安裝，用於版本管理與同步，不使用分支
- 成果需手動複製下載或透過 GitHub 網頁操作
- Python：工作場所 3.10、自用筆電 3.11
- Node.js 24：僅自用筆電
- 工作場所不額外安裝軟體，方案需在現有環境內解決

## 開發架構
- 雲端容器（claude.ai session）為主力執行環境
- 本機為輕量同步站，只跑非在本地不可的任務
- GitHub 作為中繼，串接雲端產出與本機

## Session 開場流程
每個 session 開始時依序讀取：
1. `project.md`（專案方向與原則）
2. `handoff.md`（當前狀態與下一步）

確認本 session 的單一交付目標後才開始執行。

## 偏移偵測
遇到以下三類情況立即暫停，說明觀察到的偏移，問使用者要修正方向、調整藍圖、還是這本來就是計畫的一部分：

- **目標偏移**：當前子任務不再直接服務主線目標
- **成本偏移**：某個工具或格式問題佔據比主線任務更多的討論篇幅
- **證明偏移**：開始追求「讓某技術細節能運作」而非完成功能

## Wrap-up 規則
- 任務到達自然收束點時主動建議 wrap up，使用者確認後才產出輸出
- 每個 session 結束必須產出完整 handoff.md 與 commit 指令
- History 只保留最近兩個 session，舊的壓縮後移除
- 藍圖方向改變時同步更新 project.md 並記錄原因與版本

## 互動偏好
- 回應直接，說完就停
- 有不同意見就說，不要附和
- 問題問清楚再回答，不確定就說不確定
- 提問一次只問一個問題
- 需要確認方向時主動說，不要假設後繼續
- 輸出程式碼或指令時用程式碼區塊包起來
- commit message 由 Claude 根據本次 log 產出

## 狀態確認機制
遇到以下情況，在回答前先說「⚠️ 狀態確認：xxx」再繼續：
- 回報 bug 但未說明是否使用最新版本——確認後直接從其他方向排查，不重複確認
- 同時提出兩個以上不同性質的問題
- 同一問題修超過兩輪未解決——停下來說明可能的根本原因，確認方向後再繼續
