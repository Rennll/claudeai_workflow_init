# 使用者環境與限制

## 使用平台
- claude.ai 免費版網頁（主要）及手機 App
- 無法跨 session 保留記憶，每次需貼入交接包
- 不使用 Claude 桌面版、本機 AI agent

## 本機環境
- Windows 10 + PowerShell 7
- Git 已安裝，用於版本管理與同步
- 成果需手動複製下載或透過 GitHub 網頁操作
- Python：工作場所 3.10、自用筆電 3.11
- Node.js 24：僅自用筆電
- 工作場所不額外安裝軟體，方案需在現有環境內解決

## 開發架構
- 雲端容器（claude.ai session）為主力執行環境
- 本機為輕量同步站，只跑非在本地不可的任務
- GitHub 作為中繼，串接雲端產出與本機

## 工作流規則
- 每個 session 結束前必須產出交接包與 commit 指令
- 第一個 session 只產出 project.md 初稿與第一份交接包，不進入實作
- 藍圖方向改變時必須同步更新 project.md 並記錄原因
- 發現任務偏移時立即暫停確認，不繼續往下做

## 互動偏好
- 提問一次只問一個問題
- 需要確認方向時主動說，不要假設後繼續
- 輸出程式碼或指令時用程式碼區塊包起來
- commit message 由 Claude 根據本次 log 產出
